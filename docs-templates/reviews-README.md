# Reviews

Code reviews, security audits, and architectural assessments.

## What Goes Here

- Security audits
- Performance reviews
- Architecture assessments
- Dependency audits
- Code quality reviews

## Naming Convention

Use date-prefixed names: `YYYY-MM-DD-description.md`

Examples:
- `2025-12-08-security-audit.md`
- `2025-12-08-performance-review.md`
- `2025-12-08-dependency-audit.md`

## Template

```markdown
# Review: Title

**Date:** YYYY-MM-DD
**Reviewer:** Name or "Claude"
**Scope:** What was reviewed

## Summary

High-level findings and overall assessment.

## Findings

### Finding 1: Title

**Severity:** Critical | High | Medium | Low | Info
**Status:** Open | Resolved | Accepted Risk

Description of the finding.

**Recommendation:** What to do about it.

### Finding 2: Title

**Severity:** Level
**Status:** Status

Description and recommendation.

## Positive Observations

What's working well.

## Action Items

- [ ] Priority action item
- [ ] Another action item

## Follow-up

When to re-review or next steps.
```
